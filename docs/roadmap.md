# ModelSentinel 5-Phase Roadmap

## Phase 1: Productize The Demo

Goal: turn the static HTML prototype into a clean, modern frontend app.

Deliverables:

- recreate the dashboard in React + TypeScript
- split UI into reusable components
- move hardcoded feature data into mock API responses
- make the layout responsive for desktop and laptop demos
- add screenshots and a project overview GIF for GitHub

Exit criteria:

- the UI works locally
- interactions feel polished
- the repo already looks professional to a recruiter

## Phase 2: Build The Backend API

Goal: replace browser-only state with real endpoints.

Deliverables:

- FastAPI service with typed schemas
- endpoints for feature metrics, alerts, trends, and retraining status
- simple in-memory or file-based storage for the first version
- clean API docs through Swagger / OpenAPI

Exit criteria:

- the frontend loads all data from the API
- no key dashboard metric is hardcoded in the browser

## Phase 3: Add Real Drift Logic

Goal: make the monitoring story technically credible.

Deliverables:

- baseline dataset and synthetic production batch generator
- PSI implementation for each monitored feature
- KS statistic calculation
- alert thresholds and severity classification
- drift snapshot job that updates metrics on a schedule

Exit criteria:

- drift values are generated by Python logic
- you can explain exactly how each metric is computed in an interview

## Phase 4: Add Retraining And Model Lifecycle

Goal: simulate recovery, not just detection.

Deliverables:

- training pipeline for a credit-risk model
- evaluation metrics before and after retraining
- model version registry file or metadata table
- retraining endpoint and job status flow
- updated dashboard state after a successful deployment

Exit criteria:

- the retraining button maps to a real backend workflow
- the project shows both monitoring and remediation

## Phase 5: Ship, Test, And Sell It

Goal: make the project recruiter-ready and interview-ready.

Deliverables:

- deployment of frontend and API
- seed data and demo scripts
- unit tests for PSI / KS logic
- API smoke tests
- README with architecture, screenshots, tradeoffs, and demo link
- 3 strong resume bullets and a LinkedIn-ready project summary

Exit criteria:

- a recruiter can click a live link
- a hiring manager can scan the README and understand the impact
- you can demo the project confidently in under 3 minutes
